Hello! Welcome to Lea's World. I first built this site so that others could learn more about me, but I like to also keep it as a personal record of what I've made over the years.

Before I let you look at my work since highschool, let me first introduce myself. I'm Lea Broudo, a software engineer from New York. I've worked on a range of projects, some of which I've included on this site.

Between 2014 and 2018 I mainly taught myself to draw portraits, since I was interested in being able to capture people's likeness. After portraits, I began to work on larger and more surreal scenes, as well as branch off into other mediums like embroidery and pottery. You can check out some of my work here.

While pursuing my undergrad at Columbia I learned to program, but realized I could continue art within engineering. I specialized in computer graphics and vision, and worked on VR projects at Steven Feiner's Computer Graphics and User Interfaces Lab.

Since then, I've been working on an Oculus Quest game, Holo Brawl, through the Oculus Start Program.

HoloBrawl is a VR board game that allows players to personally interact with their pieces. Pieces squirm when picked up, and have personalities of their own! The objective of the game is to kill all of your AI opponent’s pieces before they kill yours.

Players must move their pieces to an opponent’s tile to attack them. Each piece has an attack and defense score, and if the difference between the attacker’s attack score and the defense’s defense score is big enough, with a bit of chance the attacker will kill the defense piece. Given different pairings of these scores, pieces can also be counter-killed or pushed to a new tile.

I worked at Columbia University's Computer Graphics and User Interfaces (CGUI) Lab on a Collaborative Urban Virtual Environment under Professor Steven Feiner and lead Doctoral researchers. The project is used for 3D urban data visualization in VR, allowing users to select infrastructure to examine, filter, and expand data. I further developed the Unity framework with SteamVR and the Mapbox API to research the interactions of multiple users within a virtual representation of NYC. I also implemented an exclusively Desktop interface so that users without head-worn displays or hand-held mobile devices during the pandemic could still use CURVE.

Along with two classmates using Unity and Vuforia I developed sineAR, an augmented reality app that allows users to learn about acoustic waves through creating and modifying spatial audio ensembles. With step-by-step tutorials and multiple methods of interaction, users quickly and immersively create sounds in their AR environment and experience how modifying frequency and amplitude leads to an interesting new world of sound.

This is a Raytracer I wrote that uses BRDF calculations to render images. Implemented recursive reflections, depth of field, and shadows from scratch.

A number of GLSL shaders I implemented, including: Phong, Gouraud, texture modulation, normal mapping, checkerboard pattern, cel (toon), and a more creative rainbow shader!

This is a plugin I wrote for Maya that allows users to create meshes of different types of insects. These insects are created either randomly or according to user-specified input in a slider menu. The tool was written in Python using Maya's API, and the final insects were rendered in Arnold.

In this project, I was interested in creating a Maya Plugin that let users easily create different types of trees. In order to generate trees that looked realistic, I used procedural L-systems with stochastic sampling. Users can input their own seeds to create even more varied trees, and they can also control parameters like branch tapering, leaf and flower density, and mesh texturing.

The tool was written in Python for Maya, and the final trees were rendered in Arnold.

This is a character I modeled in Maya and textured using Substance Painter. I was responsible for the concept and design of the character. The character also has a fully functional rig with full IK/FK functionality for the arms, legs, and spine. The walk cycle animation was also done be me, with the final scene rendered in Arnold.

For this project, I wanted to create a tool that gave artists a realistic figure model to use for sketching or illustration. I thought it was time for an upgrade from those old-school wooden models, so Sketch Sage was born.

The UI was developed using plain JavaScript and HTML/CSS, and the graphics were all made using THREE.js. After the desktop and mobile versions were completed in July 2019, the site gained over 1,500 active users in its first four months.
